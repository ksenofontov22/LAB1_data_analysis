# Отчет о качестве данных

## 1. Введение

Датасет "hourly_weаther_data" содержит данные о метеорологических и атмосферных условиях, собираемых датчиками с переодичностью в 1 час. Сбор данных производился с 11 января 2011 года по 15 февраля 2014 года.

Заказчиком анализа может являться служба прогноза погоды, служба мониторинга окружающей среды, научные отделы и лаборатории изучающие динамику атмосферы.

С помощью этого датасета можно решить такие задачи интеллектуального анализа данных, как:
 - категоризация погодных условий местности
 - сбор статистики по частоте погодных явлений
 - отслеживание стихийных бедствий

## 2. Паспорт датасета: 

Размер датасета: 21208 строк, 12 столбцов

Признаки:  

visibility - Видимость 

windBearing - Направление ветра от севера в градусах 

temperature - Температура 

time - Дата и время измерения 

dewPoint - Точка росы 

pressure - Давление 

apparentTemperature - Ощущаемая температура 

windSpeed - Скорость ветра 

precipType - Тип осадков 

curWeather - Текущая погода 

humidity - Влажность 

summary - Итог, статус погоды на время замера 


## 3. Основные проблемы качества данных:  

| № | Тип проблемы | Описание | Потенциальный риск для анализа | Предлагаемое решение/ стратегия |
|---|--------------|----------|--------------------------------|---------------------------------|
| 1 | Пропущенные значения | В столбцах `pressure` и `curWeather` отсутствует несколько значений | Неполнота данных для анализа | Восстановление данных на основе значений замеров за предыдущие и следующие часы. |
| 2 | Дубликаты | Наличие дублирующихся данных об одном и том же промежутке времени | завышение статистики для категориальных признаков и искажение в результатах при анализе числовых | Удаление дубликатов, выявление причины дублирования данных |
| 3 | Проблемы качества категориальных данных | Признак `summary` содержит много значений с лишними пробелами в записи | Возникновение ошибок при подсчёте уникальных значений в столбце, ошибки в гистаграммах по данному признаку | Стандартизация строк с помощью интсрументов языка Python |

## 4. План улучшения качества:

**Высокий приоритет:**

- Удаление дублирующихся строк.
- Удаление начальных и конечных пробелов в записи категориальных данных.

**Низкий приоритет:**

- Добавление новых категорий погодных условий
- Разграничение осадков категории "дождь" на более информативные (учёт плотности осадков, риска грозы)

## 5. Краткое резюме
В процессе работы научился описывать датасет, выполнять первичный обзор данных, строить базовые графики для визуализации, выявлять ошибки в данных и предлагать пути их решения, работать в системе Jupyter Notebook.  
Особое внимание при анализе погодных условий и явлений стоит уделить выявлению закономерностей между параметрами, например - температурой и влажностью воздуха, а также влиянию географических условий местности на получаемые данные.
